Makaseh Public Decryption Challenge

This challenge invites independent verification of Makaseh Identity’s security model.

What you are allowed to do

  • Export encrypted payloads from the Makaseh demo
  • Store, share, and analyze ciphertext freely
  • Use any third-party cryptographic or forensic tools
  • Attempt brute-force, pattern analysis, or correlation

What this challenge is not

  • No hidden keys are provided
  • No passwords or credentials exist
  • No backend systems are exposed
  • No identity data is embedded

Success criteria

If meaningful plaintext, identity, or reusable structure can be recovered from exported payloads without a valid Makaseh token, the system has failed.

Expected outcome

Ciphertext has no standalone value. Possession alone grants no access, insight, or leverage.

Important note

All demo data is synthetic and non-personal. This challenge is provided solely for evaluation and education. No production systems are involved.

Makaseh principle:
Security is architectural, not policy-based.
